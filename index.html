2019年 在校独立开发

  • 项目描述 :
    • 业务介绍：一个用于共享快递信息，为校友方便快递“代领”的信息共享平台。
    • 后端技术：SpringBoot（容器）、SpringSecurity（安全框架）、JWT（身份验证方案）、Mybatis（ORM框架）、Mapper（Mybatis辅助插件）、Redis（缓存数据库）、FastDFS（文件系统）
    • 前端技术：wepy（小程序开发框架）、MinUI（UI框架）
  • 项目问题及解决方案 :
    1. 项目最初基于SpringSecurity的sessionID作为Cookie来保持用户状态；但发现苹果手机无法登录；结合JWT的身份验证方式解决了该问题。
    2. 在多个人同一时间接收同一个订单时，违背了一个订单只能被一人接收的规则。利用Redis键的唯一性特征解决了问题——当一个订单被用户接收时，先检测MySql数据库该订单的状态；如果为未被接收状态，则检测Redis内有没有以该订单ID作为Key值的键；如果没有，将该订单ID作为Key值存入Redis；再进行更改数据库内该订单的状态、发送短信通知等操作；待该事务完成后，删除掉该Key值。
    3. 身份认证更换为JWT方式后，关闭了Spring对用户状态（session）的维护；导致原本以session为生命周期范围的短信验证码服务无法正常工作（原本bean以session为生命周期，并创建一个线程5分钟后清除验证码值）。最后利用Redis的限时键值对功能，将手机号作为Key，验证码作为Value，有效时间为5分钟。
VC后台管理系统

2019-至今 天才纵横国际企业管理(北京)有限公司 Java 后端开发工程师/前端开发工程师

  • 项目描述 :
    • 业务介绍：该项目是管理端系统，目的是对C端用户及业务数据的统计、审核、管理等操作便捷化。
    • 后端技术：SpringBoot（容器）、SpringSecurity（安全框架）、Mybatis（ORM框架）、Mybatis Plus（Mybatis辅助插件）、Redis（缓存数据库）
    • 前端技术：Vue（小JavaScript框架）、elementUI（UI框架）
  • 工作内容 :
    1. 由于业务要求该系统需要从已有的管理系统携带分配的权限跳转至该系统，且该系统保持独立。我对SpringSecurity的登录方式进行重写，由原来的帐号密码登录改为Key值登录。老系统将该用户对象序列化作为Value存入Redis，以随机生成的字符串作为Key，并将该字符串返回至前端页面；前端页面携带加密后的字符串跳转至新系统并将字符串作为Key值进行登录，新系统后端以解密后的字符串作为Key值从Redis获取Value，对Value值反序列化后获取用户对象，将该对象交给SpringSecurity管理即可，以此解决这一业务需求。
    2. 该系统为前后端分离模式，业务需求根据角色实现动态菜单。我重写SpringSecurity登录成功后的行为，登录成功后将该用户的所有权限标识返回至前端；前端接受后通过不同的权限标识进行渲染菜单。
    3. 业务要求保证每个前端页面的访问必须拥有该权限。我将每个页面的访问权限标识作为路由标识，通过前置路由守卫对访问对当前用户拥有的权限进行拦截校验，解决了这一问题。
